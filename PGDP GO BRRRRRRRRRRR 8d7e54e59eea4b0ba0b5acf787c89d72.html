        static int findGCD(int a, int b) {
        if (b == 0) {
            return a;
        } else {
            // Subtract the smaller number from the larger number
            if (a > b) {
                return findGCD(a - b, b);
            } else {
                return findGCD(a, b - a);
            }
        }
    }

        } else if (decimal == 1) {

        private static boolean isArrayContainsValue(int[] array, int value) {
        return isArrayContainsValueHelper(array, value, 0);
    }

    private static boolean isArrayContainsValueHelper(int[] array, int value, int index) {
        // Base case: If the index is equal to the array length, the value was not found
        if (index == array.length) {
            return false;
        }

        // Check if the current element is equal to the desired value
        if (array[index] == value) {
            return true;
        }

        // Recursive case: Continue the search in the rest of the array
        return isArrayContainsValueHelper(array, value, index + 1);
    }

        static int BinarySearch (int[] arr, int low, int high, int target){
        if(high>=low){
            int mid = (low + high) / 2;
        if (arr[(mid)] == target){
            return mid;
        }
        if (target > arr[mid]){
            return BinarySearch(arr, mid+1, high, target);
        }
        if (target < arr[mid]){
            return BinarySearch(arr, low, mid-1, target);
        }
        }
        return -1;
    }

        Finding first occurrence:

        private static int binarySearchRecursive(int[] array, int target, int left, int right) {

        if (left>right){
            return -1;
        }
        int t = left+( right-left)/2;
        if (array[t] < target) return binarySearchRecursive(array, target,t+1, right);
        int t0 = binarySearchRecursive(array, target, left, t-1);
        if (t0 >=0) return t0;
        if (array[t] == target) return t;
        else return -1;
    }

  • Classes & Objects
    • Access Modifiers

      There are four types of Java access modifiers:

      1. Private: The access level of a private modifier is only within the class. It cannot be accessed from outside the class.
      1. Default: The access level of a default modifier is only within the package. It cannot be accessed from outside the package. If you do not specify any access level, it will be the default.
      1. Protected: The access level of a protected modifier is within the package and outside the package through child class. If you do not make the child class, it cannot be accessed from outside the package.
      1. Public: The access level of a public modifier is everywhere. It can be accessed from within the class, outside the class, within the package and outside the package.
    • Final Keyword

      In Java, the final keyword can be applied to classes, methods, and variables, and its meaning varies depending on where it is used.

      1. Final Classes:
        • When applied to a class, the final keyword indicates that the class cannot be subclassed. In other words, it prevents other classes from inheriting from it.
        • Example:
          final class FinalClass {
    // Class members and methods
}
      1. Final Methods:
        • When applied to a method, the final keyword indicates that the method cannot be overridden by subclasses. Subclasses are not allowed to provide a different implementation for a final method.
        • Example:
          class Parent {
    final void finalMethod() {
        // Method implementation
    }
}

class Child extends Parent {
    // Compilation error: Cannot override final method from Parent
    // void finalMethod() { }
}
      1. Final Variables (Fields):
        • When applied to a variable (field), the final keyword indicates that the variable's value cannot be changed after it has been initialized. Once assigned, a final variable becomes a constant and cannot be reassigned.
        • Example:
          class MyClass {
    final int myConstant = 10;

    void modifyConstant() {
        // Compilation error: Cannot assign a value to final variable 'myConstant'
        // myConstant = 20;
    }
}

      In summary:

      • For classes, final prevents inheritance (sub-classing).
      • For methods, final prevents overriding in subclasses.
      • For variables (fields), final makes the variable a constant, and its value cannot be changed after initialization.

      Using final appropriately can lead to more robust and maintainable code by enforcing constraints that help prevent unintended modifications or extensions in the codebase.

    • Real Life use of Downcasting

      Consider a real-life scenario where downcasting can be useful: a system managing different types of employees in a company.

      class Employee {
    String name;

    public Employee(String name) {
        this.name = name;
    }

    public void work() {
        System.out.println("Performing general tasks.");
    }
}

class Developer extends Employee {
    String programmingLanguage;

    public Developer(String name, String programmingLanguage) {
        super(name);
        this.programmingLanguage = programmingLanguage;
    }

    @Override
    public void work() {
        System.out.println("Writing code in " + programmingLanguage);
    }

    public void codeReview() {
        System.out.println("Conducting code review.");
    }
}

class Manager extends Employee {
    public Manager(String name) {
        super(name);
    }

    @Override
    public void work() {
        System.out.println("Managing team and projects.");
    }

    public void conductMeeting() {
        System.out.println("Conducting a team meeting.");
    }
}

      In this example, we have a hierarchy of employees, where Developer and Manager are subclasses of the general Employee class.

      Now, let's create a method that works with an array of employees and performs different actions based on their types:

      public class Company {
    public static void main(String[] args) {
        Employee[] employees = new Employee[3];
        employees[0] = new Developer("Alice", "Java");
        employees[1] = new Manager("Bob");
        employees[2] = new Developer("Charlie", "Python");

        for (Employee employee : employees) {
            employee.work();

            // Downcast to access specific methods based on the type
            if (employee instanceof Developer) {
                Developer developer = (Developer) employee;
                developer.codeReview();
            } else if (employee instanceof Manager) {
                Manager manager = (Manager) employee;
                manager.conductMeeting();
            }
        }
    }
}

      In this scenario, the work method is overridden in both Developer and Manager classes to provide specialized behavior. During the iteration over the array of employees, we downcast each Employee to its specific type using instanceof to access methods or properties unique to that type.

      This reflects a situation where you have a generic way of handling employees, but sometimes you need to access specialized functionalities specific to certain types of employees.
